1. What the plugin does

AI Blog Automator connects WordPress to large language models (Gemini by default, with optional OpenAI, Anthropic Claude, and custom OpenAI-compatible APIs). It generates long-form HTML articles from topics and keywords, adds FAQ blocks, attaches stock images, injects internal links, writes SEO meta for supported plugins, and optionally notifies Google’s Indexing API.

2. Requirements and API keys

  • WordPress 6.0+ and PHP 8.0+.
  • At least one LLM key (Gemini recommended).
  • For reliable images: Pexels API key and/or Unsplash Access Key (official API; legacy Unsplash Source URLs are not used).
  • Optional: Google service account JSON for the Indexing API.
Store production keys securely. On shared hosting, restrict file permissions on backups that include the database (options table holds keys unless you use constants in wp-config.php).

3. Installation and activation

  1. Copy the plugin folder into wp-content/plugins/ (preserve ai-blog-automator.php at the plugin root).
  2. In wp-admin: Plugins → Installed Plugins → Activate “AI Blog Automator”.
  3. Activation registers schedules and creates queue/log database structures as needed.
  4. Open AI Automator → Settings, enter keys, click Save, then use Test API connections.

4. Admin menu overview

The plugin adds a top-level AI Automator item in the WordPress admin sidebar. Submenus typically include Dashboard, Settings, Generate, Queue, and Activity logs (exact labels may vary with translations).

5. Settings walkthrough

The API tab is where you connect external services. After saving, use Test API connections to verify Gemini, optional providers, Pexels, Unsplash, and indexing credentials.

Other tabs (summary)

  • Content — niche, word count, tone, language, author, default categories, images per post, max internal links.
  • Prompts & formats — article template slug; optional outline/section prompt prefix, suffix, and global append.
  • Automation — posts per day, publish time, publish status, queue recurrence.
  • SEO — plugin detection vs native; FAQ and Article schema toggles.
  • Advanced — log retention, disabled post types for front-end meta output.

6. Generate a single post

Use AI Automator → Generate for a one-off article. Fill topic/title and primary keyword, add optional secondary keywords and categories if shown, then submit. The browser runs an AJAX request; when complete, follow the link to edit the new draft.

Always review AI output for factual accuracy, licensing of cited sources, and broken or hallucinated URLs before publishing.

7. Queue: single row and bulk import

Single “Add to queue” creates one pending job with topic, focus keyword, optional other keywords, categories, and article format.

Bulk queue accepts many lines at once—ideal for a month of titles from a spreadsheet.

Column reference (bulk)

  • Title — post topic / working title.
  • Focus keyphrase — primary SEO keyword for that row.
  • Other keywords — comma or semicolon separated inside the cell/field.
  • Category IDs — WordPress category term IDs, comma-separated (optional; defaults apply if empty).
  • Article format — template slug (optional).

Legacy lines Title|keyword or a single title per line still work.

8. Automation and cron

Queue processing relies on WordPress cron unless your host triggers wp-cron.php on a system schedule. Low-traffic sites should use real cron or a host feature so pending jobs run on time. “Publish time” in Automation combines with bulk stagger dates to set scheduled_at on queue rows.

9. Images (Pexels and Unsplash)

Featured and in-content images try Pexels first when a key is set, then Unsplash via the official API when an Access Key is set. Without keys, downloads fail and errors appear in Activity logs. Captions and credits are stored where the APIs provide photographer metadata.

10. SEO plugins

When Yoast, Rank Math, or All in One SEO is active (or set explicitly in Settings), the plugin writes focus keyword, meta description, title where supported, and maps secondary keywords into synonyms / additional keywords / combined keyword fields per integration. Native mode stores _aiba_* meta and can output tags when no SEO plugin handles them.

11. Internal links

During generation, the model may insert machine tokens such as [INTERNAL_LINK_PLACEHOLDER: anchor phrase]. The internal linker replaces all variants with real <a href> links to published posts or pages, or sensible fallbacks, so visitors never see raw placeholder text.

12. Activity logs

Use the plugin’s log screen to diagnose LLM errors, HTTP failures for images, indexing API responses, and queue processing. Increase retention in Advanced if you need longer history (within reasonable database size).

13. Troubleshooting

  • Generate button does nothing — check browser console; confirm scripts load; verify you are on the correct admin.php?page=... screen.
  • 429 / rate limit — reduce word count or daily volume; add fallback API keys in Auto mode.
  • No images — add Pexels and/or Unsplash keys; run API test.
  • Queue stuck — verify cron; read Activity logs for SQL or capability errors.
