# Screenshot checklist for CodeCanyon

Envato’s **exact pixel sizes** change; confirm under **Author Help → WordPress upload requirements** before export.

Typical needs:

- **Preview image(s)** — e.g. 590×300 JPEG (main feature image).
- **Optional inline screenshots** in the description (same or larger).

## Recommended captures (export @2x then scale down for sharpness)

1. **Dashboard** — `AI Automator → Dashboard` with stat tiles and recent activity visible.
2. **Generate** — form with topic/keyword fields and primary button.
3. **Queue** — table + bulk textarea area (one example line visible).
4. **Settings → API** — Gemini + optional keys (blur or fake keys in screenshot).
5. **Settings → Content** — word count / tone / images per post.
6. **Settings → Automation** — posts per day, publish time.
7. **Settings → SEO** — plugin mode + schema toggles.
8. **Logs** — striped table with a few rows.
9. **Upgrade** — free vs premium columns + CTA buttons.
10. **Feedback** — form (no real user messages if sensitive).
11. **Welcome notice** — optional: capture once after fresh install.
12. **Published post** — front-end result: featured image + FAQ block (optional).

## Naming (example)

- `01-dashboard.jpg`
- `02-generate.jpg`
- … through `10-feedback.jpg`

## Tips

- Use a **clean admin profile** (no unrelated notices).
- **English UI** for broadest buyer appeal unless you market a localized demo.
- **Blur API keys** even if fake.
