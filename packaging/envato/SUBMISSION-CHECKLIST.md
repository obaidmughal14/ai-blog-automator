# Pre-submission checklist (Envato WordPress)

## Code & zip

- [ ] Zip contains **only** the plugin folder (e.g. `ai-blog-automator/`) at root, **not** nested `ai-blog-automator/ai-blog-automator/`.
- [ ] No `node_modules`, `.git`, `.env`, or local IDE junk inside the zip.
- [ ] `readme.txt` **Stable tag** matches `Version` in main PHP file.
- [ ] **Tested up to** matches what you actually smoke-tested.
- [ ] No `eval`, no remote code inclusion, no unchecked `$_GET`/`$_POST` on front end.

## Documentation

- [ ] **INSTALLATION.md** (this pack) matches your live process.
- [ ] **ITEM-DESCRIPTION.html** placeholders replaced (`YOUR-SUPPORT-URL`, `YOUR-DEMO-URL`).
- [ ] **CREDITS-THIRD-PARTY.md** + **PRIVACY-DATA.md** linked or pasted in documentation tab.

## Assets

- [ ] **Preview image** exported per Envato current size rules.
- [ ] **Screenshots** from `SCREENSHOTS-CHECKLIST.md` (minimum count per current rules).

## Legal

- [ ] **Support policy** finalized (`SUPPORT-POLICY.md`).
- [ ] **GPL** compatible distribution; Envato **split license** acknowledged where required.

## After approval

- [ ] Pin a **“how to get your API keys”** FAQ on your support channel.
- [ ] Keep **CHANGELOG-BUYERS.md** in sync for each Envato update.
